Примеры моих проектов

1. Программный комплекс для загрузки объявлений на avito.ru, youla.ru, jazi.ru

(Авито - крупнейшая в России доска для размещения объявлений. Почти каждый в России пользуется Авито)
Что умеет моя программа(отдельные модули, связанные в одно):
1-1. Получение и обработка xml любых подвидов от клиента. Обновление раз в час
Стек технологий: xml.etree.ElementTree,bs4,codecs,urllib.request
1-2. Автоматическая выгрузка данны в Google Spreadsheets с изменением данных на лету (распознавание категорий товаров, редактирование названий, описаний, стоимости и других параметров)
Стек технологий: google.oauth2, googleapiclient
После реализации этого модуля, я понял, что гугл апи имеет сильное ограничение на количество запросов. И позже я еще доработал переключение между разными аккаунтами.
1-3. оздание xml файлов с данными из Google Spreadsheets для каждого клиента
Стек технологий: pandas, gspread, gspread_dataframe
1-4. Автоматическая генерация текста с наложением на них на лету при формировании фида раз в час
Стек технологий: cv2, numpy, PIL
1-5. Автоматическое изменение параметиров фотографий из фида
Стек технологий: PIL
1-6. Автоматическое создание коллажей и накладывание изображение на главное фото
Стек технологий: PIL
1-7. Автоопределение фона, вырезание товара и накладывания на иной фон (опять же на лету)
Стек технологий: PIL, cv2
1-8. Загрузка и чтение фтп. Разные фтп для разных клиентов
Стек технологий: ftplib, io

Краткое резюме проекта
Изначально этот проект создавался как простейшая автоматизация для одного крупного клиента.
Но функциона постоянно разрастался и превратился в программный комплекс, с помощью которого
компания ведет сотни клиентов (таких крупных, которых не могла бы получить без автоматизации).
Архитектура постоянно менялась. И всегда не было времени, чтобы улучшать ее, переписывать код.
Большинство разработческих задач ставились в тот момент когда они нужны еще вчера.
Новые разработки всегда были в приоретете перед оптимизацией старого кода.
Однако я старался комментировать код и делать его таким, чтобы любой другой разработчик
смог в нем разобраться. Не смотря на то, что это первый мой такой крупный проект,
разработанный только своими силами.

2. Инструмент анализа выдачи Авито avito.ru

GitHub code
Я могу показывать этот код, потому что делал для себя.
Он анализирует выдачу Авито по запросам в нужных регионах и выгружает данные в гугл таблицу.

4. Телеграм бот, позволяющие отвечать клиентам Авито прямо в телеграме avito.ru

В этом проекте я использовал протейший сервер flask и sqlite.
К сожалению, работа с базами данных мне нужна была очень редко,
потому что Google Sheets было идеальным хранилищем данных, с учетом того,
что пользователям нужен постоянный доступ к информации. Это позволило не
разрабатывать отдельный интерфейс для взаимодействия пользователями с данными.

В этом блоке у меня возникало очень мнго проблем с Авито апи -
он был сделал недавно и не всегда работал (что сильно осложняло тестирование).
Также я сталкивался с большим количеством незадокументированных ошибок в работе Авито апи.
Стек технологий: requests, json, flask, sqlite3, telethon (telegram client)
Ранее, 2-3 года назад я писал порядка 10 ботов для телеграма с клавиатурами и полезным функционалом.

5. АПИ интеграция с Bitrix24 и Amocrm

5-1. Выгрузка данных в гугл таблицы, чтобы делать отчеты для бизнеса с большие кастомизацией
5-2. Выгрузка и загрузка данных для переноса в другую crm/erp (обычно из amocrm в b24)
5-3. Интераграция со сторонними нестандартными сайтами
5-4. Интеграция с внешней телефонией (только информация о звонках, не сами звонки)
5-5. Более глубокая автоматизация внешними разработками, которая недоступна в стандартном функционале
Стек технологий: pandas, gspread, gspread_dataframe, requests, json
